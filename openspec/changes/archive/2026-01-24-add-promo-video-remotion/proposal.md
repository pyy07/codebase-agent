# Change: 使用 Remotion 制作推广视频

## Why

项目需要一个专业的推广视频来展示 Codebase Driven Agent 的核心价值和功能特性。推广视频可以帮助：

1. **提升项目知名度**：通过视觉化的方式向潜在用户展示项目的核心能力
2. **降低理解门槛**：视频比文字文档更直观，能够快速传达项目的价值主张
3. **增强用户信任**：专业的推广视频能够提升项目的专业形象
4. **便于分享传播**：视频格式便于在社交媒体、技术社区等平台分享

选择 Remotion 的原因：
- **React 生态集成**：项目前端已使用 React，Remotion 基于 React，技术栈一致
- **代码驱动视频**：符合项目"代码驱动"的理念，视频内容可以通过代码版本控制
- **灵活性和可维护性**：视频内容可以像代码一样进行版本管理、复用和迭代
- **高质量输出**：支持 4K、60fps 等高质量视频输出

## What Changes

- **新增 Remotion 视频项目**：
  - 在项目中创建独立的 Remotion 视频项目目录
  - 配置 Remotion 开发环境和构建流程
  - 集成到现有的前端构建系统中

- **视频内容制作**：
  - 创建展示项目核心功能的视频场景
  - 设计动画效果展示 Agent 工作流程
  - 制作项目 Logo 和品牌元素动画
  - 添加文字说明和关键信息展示

- **视频导出和部署**：
  - 支持导出多种格式（MP4、WebM 等）
  - 支持多种分辨率和质量设置
  - 提供视频预览和开发工具
  - 集成到项目文档或 README 中

- **开发工具和脚本**：
  - 提供视频开发、预览、渲染的命令行工具
  - 支持快速迭代和测试
  - 文档化视频制作流程

## Impact

- **受影响的规范**：`promo-video` capability（新增）
- **受影响的代码**：
  - `web/` - 新增 Remotion 相关配置和依赖
  - `web/src/video/` - 新增视频组件和场景（新建目录）
  - `web/remotion.config.ts` - Remotion 配置文件（新建）
  - `package.json` - 新增 Remotion 相关依赖和脚本
  - `README.md` - 更新文档说明如何生成视频
  - `.gitignore` - 添加视频输出文件忽略规则

- **向后兼容性**：完全兼容，不影响现有功能。Remotion 视频项目作为独立模块存在，不影响主应用的构建和运行。

- **新增依赖**：
  - `remotion` - Remotion 核心库
  - `@remotion/bundler` - Remotion 打包工具
  - `@remotion/cli` - Remotion 命令行工具
  - 其他 Remotion 相关依赖（根据需求添加）
